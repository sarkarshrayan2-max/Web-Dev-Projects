# Chroma Match Puzzle // Colometric Variance Analytics

Chroma Match Puzzle is an advanced visual perception trainer built entirely client-side to resolve the color mutation and rapid timing constraints defined in tracking item **Issue #876**. By generating closely mirrored decoy parameters inside the Hue-Saturation-Lightness (HSL) geometric space, the system tests and maps chromatic acuity under intense time limits with zero third-party framework layers.

---

## Technical Capabilities & Core Specifications

### 1. Minimalist Interface Architecture (`style.css`)

- **Strict Real Estate Viewport Constraints:** Confines all control rows, grid choices, and score panels inside an anti-shiver chassis foot-print (`max-width: 1200px`, `height: 760px`) to prevent visual fragmentation over variable desktop monitors.
- **Jitter-Proof Monospaced Typography Locks:** Active countdown timelines, streak counters, historical record numbers, and color hexadecimal string characters utilize explicit monospaced font declarations (`ui-monospace, Consolas`), guaranteeing structural layout points stay perfectly locked during numeric mutations.
- **Tactile Component Shaders:**
  - `.target-specimen-display`: Centered large reference tile displaying a pseudo-random hex array matching real-time alpha ranges.
  - `.choice-card`: Interactive selection grids hosting subtle, responsive border transformations on hover alongside hardware-accelerated click indicators.
  - `.progress-timer-track`: Features an unyielding fluid animation loop that shrinks linear parameters cleanly down to zero over 5 seconds.

### 2. Procedural Color Mutation Logic (`script.js`)

- **HSL Variance Mutation Algorithm:** Rather than executing broad random RGB generations, the codebase transforms base colors into Hue, Saturation, and Lightness coordinates ($H \in [0,360], S \in [0,100], L \in [0,100]$). It constructs decoy variations by applying precise micro delta offsets ($\Delta h, \Delta s, \Delta l$) to prevent immediate visual distinction. As player streaks escalate, standard deviation margins shrink systematically to raise sensory difficulty.
- **Fisher-Yates Positioning Scrambler:** Maps response arrays through a random distribution pass each round to scramble the correct slot completely across the selection panel, breaking muscle memory adaptations and index repetition habits.
- **High-Frequency Countdown Thread:** Employs a precise ticking time loop managing round intervals. Players are constrained to a unyielding 5-second response window per round; failing to log an interactive selection prior to clock expiration triggers an instant full game-over state cascade.
- **LocalStorage Session Retention:** Persists and increments maximum score ceilings natively inside client data tracks to ensure user record profiles survive window refreshes.
